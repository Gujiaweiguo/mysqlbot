## 1) 结论摘要

当前 `mallcre` 五类主题的相关表都已经勾选，但测试数据强弱不均：

| 主题 | 结论 | 说明 |
|---|---|---|
| 商铺出租情况 | PASS | 现有数据足够支撑空置、在租、楼层分布类分析 |
| 商户情况 | PASS | 已补齐商户维表，可做基础画像与商户-铺位-合同关联查询 |
| 合同情况 | PASS | 已补齐项目级合同维表，可做合同明细、数量和覆盖率分析 |

| 收缴情况 | PARTIAL | 已能支撑付款方式分布，暂不适合复杂收缴率链路分析 |

## 4) 已验证可用的提问模板

以下问法已经有回归脚本或真实数据验证支撑，优先推荐使用。

### 4.1 商铺出租情况

1. 请列出星河购物中心当前空置铺位的楼层、铺位名称和面积。
2. 请统计星河购物中心空置铺位在各楼层的数量分布，并按空置数量从高到低排序。
3. 请统计星河购物中心当前已租铺位和空置铺位的数量对比。

### 4.2 商户情况

1. 请列出 `STORE_DEMO_001` 当前商户名单及联系人信息。
2. 请统计 `STORE_DEMO_001` 当前商户数量，并按商户简称展示。
3. 请列出 `STORE_DEMO_001` 商户的商户名称、简称、联系人和联系电话。
4. 请列出星河购物中心商户与铺位、合同编号的对应关系。

### 4.3 合同情况

1. 请列出星河购物中心全部合同编号、商户名称、铺位名称、合同开始日期和结束日期。
2. 请统计星河购物中心当前已签合同的铺位数和对应商户数。
3. 请列出星河购物中心每份合同对应的商户、铺位和店招信息。

### 4.4 应收情况

1. 请统计星河购物中心2024年6月应收总额、已收总额、未收总额。
2. 请统计星河购物中心2024年6月各费用科目的应收总额、已收总额、未收总额。
3. 请列出星河购物中心2024年6月欠款金额最高的5个商户，显示商户名称、欠款金额和费用科目。
4. 请列出星河购物中心2024年6月已到最后缴款日但仍未收清的账单，显示商户、应收金额、已收金额、未收金额和最后缴款日。
5. 请列出星河购物中心2024年6月存在欠款且已逾期的商户，并显示欠款金额。

### 4.5 收缴情况

1. 请统计星河购物中心2024年6月各付款方式的收款金额，并按金额从高到低排序。
2. 请统计星河购物中心2024年6月各付款方式的收款笔数。
3. 请对比星河购物中心2024年6月支付宝、微信支付、银行卡三种方式的收款金额。

## 5) 提问时的约束建议

为提高当前测试数据下的命中率，建议遵循以下规则：

1. 尽量显式带上项目条件：`星河购物中心`
2. 商户基础信息主题优先使用：`STORE_DEMO_001`
3. 商户-合同-铺位联查主题优先使用：`星河购物中心`
4. 优先问：
   - 当前明细
   - 当前汇总
   - Top 排名
   - 分类分布
5. 暂时避免问：
    - 同比/环比
    - 长周期趋势
    - 合同到期/续签深度分析
    - 商户画像深度分析
    - 收缴率/核销链路/到账时效

## 6) 当前最大缺口

1. 合同状态当前只有单一值 `CONT_STATE='1'`
2. 当前样本量仍然偏小，更适合“当前状态分析”，不适合趋势结论
3. 商户与合同主题已可用，但还不适合做分层洞察与异常判断

## 7) 使用建议

- 如果目标是验证 NL2SQL 主链路，优先使用：**商铺出租情况 + 应收情况**
- 如果目标是验证商户主数据读取，可使用：**商户名单 / 联系人 / 数量 / 商户-铺位-合同映射**
- 如果目标是验证合同分析，可使用：**合同明细 / 合同数量 / 合同覆盖率 / 合同关联关系**
- 如果目标是验证收缴场景，优先使用：**付款方式分布**
